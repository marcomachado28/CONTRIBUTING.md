# Guia de Contribuição

## Regras
- Use nomes claros para commits.
- Sempre crie uma branch para novas funcionalidades.
- Abra PR explicando o que foi alterado.

## Estilo de código
- Use identação de 4 espaços.
- Nomeie variáveis de forma clara.

## Testes
- Antes de enviar PR, compile o projeto localmente.
